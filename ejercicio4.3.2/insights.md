En este ejercicio observamos la creación de un servidor que mantiene un estado interno (la función matemática actual) para procesar las solicitudes del cliente. El `TrigServer.java` escucha en el puerto 35000 y por defecto aplica el Coseno a los números recibidos. El `TrigClient.java` envía mensajes y recibe el resultado calculado. Si se envía el comando `fun:sin`, `fun:cos` o `fun:tan`, el servidor actualiza la operación para futuras solicitudes. Todo ello demuestra una comunicación dinámica y con mantenimiento de estado entre cliente y servidor.
