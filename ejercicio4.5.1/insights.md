En este ejercicio se mejora el Servidor Web (ejercicio anterior) para que pueda manejar múltiples solicitudes consecutivas utilizando un bucle `while(true)`. Además, el servidor es capaz de leer la ruta que pide el cliente en el método `GET`, buscar el recurso correspondiente en una carpeta `public/` y devolver los bytes del archivo junto con el header `Content-Type` adecuado (ya sea HTML, texto, o imágenes PNG/JPEG). Esto demuestra cómo se sirven recursos estáticos de forma secuencial en Java mediante sockets puros.
