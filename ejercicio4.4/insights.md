En este ejercicio se implementa un Servidor Web rudimentario (`HttpServer.java`) que escucha en el puerto 35000. Al recibir cualquier petición (como las enviadas por un navegador web al ingresar a http://localhost:35000), el servidor lee e imprime en la consola los headers de la solicitud HTTP. Luego, responde con un String estático que contiene un formato HTTP válido y una página HTML simple. Este ejercicio sirve para entender cómo los navegadores se comunican mediante HTTP utilizando Sockets TCP estándar.
